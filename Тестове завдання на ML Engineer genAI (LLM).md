## Розробка алгоритму автоматичної сегментації мобільних додатків на субніші

⏱ **Орієнтовний час виконання:** ~4 годин

### 1. Опис завдання

Ваша мета — розробити алгоритм для автоматичного розподілу мобільних додатків на вузькі субніші, використовуючи текстові дані та візуальні атрибути (ASO). Сегментація має бути настільки гранулярною, щоб усі додатки всередині однієї субніші були **прямими конкурентами**.

**Бізнес-ціль:** Автоматичне виявлення нових конкурентів для заміни мануального дослідження (research) продуктів.

### 2. Вхідні дані

Датасет містить **4200 додатків**. Для кожного додатка доступні:

1. trackName - назва Апки
2. description - опис Апки з App Store
3. screenshotUrls - посилання на АСО-скріншоти з App Store
4. overview - згенерований AI опис апки на основі description + АСО-screenshots
5. features - згенерирований AI набір фічей на основі description + АСО-screenshots

trackName, description, screenshotUrls - сирі дані.

overview, features - згенеровані нами дані на основі description + АСО-screenshots.

Ти можеш використовувати наші дані, або (якщо є натхнення) парсити додаткові дані, створювати через LLM/vLLM різні фічі на основі сирих даних, робити рерайт існуючих даних і тд (але це обов`язково, і якщо є на це додатковий час).

**Посилання на датасет:** [Google Drive Folder](https://drive.google.com/drive/folders/1TrcQUjANZXnQrpCyKp4s6WddCzTjlQTH?usp=sharing)

### 3. Вибір підходу

Тут все залежить від тебе - обирай підхід, алгоритми і ідею, яку ти вважаєш за потрібну. Можеш використовувати лише текстовий опис, можеш додавати аналіз ASO-скріншотів. Додаткові дані не рекомендуємо використовувати (на нашій практиці вони лише погіршують якість), але якщо дуже хочеш - можеш.  
  
4. Вимоги до результату (Формат)

Результат має бути представлений у вигляді JSON з назвою ніші, описом ніші та списком конкурентів для цієї ніші. **Важливо:** Правильно вибрати назву та коректно описати опис ніші. Назва/опис ніші має бути максимально точною та відображати суть продуктів всередині цієї ніші.

**Приклади правильної гранулярності:**

- **AI Interior Design:** (Home AI, ReRoom, Remodel AI) — *Прямі конкуренти.*
- **Phone Cleaner:** (Cleaner Guru, CleanUP, KeepClean) — *Прямі конкуренти.*
- **AI Notetakers:** (Summary: AI Meeting Note Taker, Minutes: AI Meeting Note Taker) — *Прямі конкуренти.*

### 5. Очікувані результати та метрики

Оскільки це завдання на пошук структури в даних, ми оцінюємо:

1. **Гранулярність:** Основний критерій це щоб всі конкуренти були із однієї суб-ніші.
2. Додатково оцінюємо якість назви і опису.  
  
6. Звіт
  Подати структурований звіт, що містить:
  1. **Код рішення:** з коментарями щодо логіки обробки даних.
  2. **Опис підходу:** чому обрано саме цю модель/алгоритм (важливо детально розписати чому використовується та чи інша модель, той чи інший алгоритм, та чи інша ідея!).
  3. **Аналіз результатів:** вивантаження (CSV/JSON) з фінальним розподілом: `AppName | Description | Sub-niche Name`.
  ### 7. Формат здачі
  Кандидат надсилає:
  - Посилання на звіт, код, результати.
  - Резюме по виконаній роботі.  
    
  Поради для оформлення:
    - **Додай візуал:** Якщо зробиш графік кластерів (через t-SNE або UMAP), а потім інтерактивний графік через plotly - це додасть +100 швидкості валідації твого рішення.
    - **Обґрунтуй вибір моделі.**
    - **Акцент на "прямих конкурентах":** Це головний біль продактів, тому покажи, що твій алгоритм відрізняє "Фоторедактор взагалі" від "AI Headshot Generator".
